# Auto-Commit Skill

When this skill is activated, follow these steps strictly:

## 1. Check Repository Status

Run `git status --porcelain` to see the current state of all files.

## 2. Stage Changes (if needed)

- If there are **unstaged changes** (files with ` M`, `??`, etc.): run `git add -A`
- If there are **only staged changes** (files with `M `, `A `, etc.): proceed directly to commit
- If there are **no changes at all**: inform the user and exit

## 3. Analyze Changes

Run `git diff --cached --stat` for a summary, then `git diff --cached` for details.

Identify and summarize:
- **Files modified/added/deleted** — list the key files affected
- **Core changes** — what logic, UI, or functionality was added/changed
- **Purpose** — why these changes were made (infer from context)

## 4. Generate Commit Message

Use the **Conventional Commits** format with an explanatory body:

```
[type]: [concise summary]

- [key change 1]
- [key change 2]
- [key change 3]
```

**Types**: `feat` (new feature), `fix` (bug fix), `refactor` (code cleanup), `docs` (documentation), `style` (formatting), `test` (tests), `chore` (maintenance)

**Subject line rules**:
- Maximum 72 characters
- Use imperative mood ("add" not "added")
- Be specific but concise

**Body rules**:
- Bullet the key changes (3-7 bullets typically)
- Explain *what* changed and *why*
- Omit obvious or trivial details

## 5. Message Constraints

- **NO AUTHOR INFO:** Do not include names, emails, "generated by AI," "Co-Authored-By," or signatures
- **NO MARKDOWN:** Do not use backticks, code blocks, or formatting in the message
- **CLEAN SYNTAX:** Ensure the message is plain text, properly escaped for the shell

## 6. Execute Commit

Run the commit command with a heredoc for proper multi-line formatting:

```bash
git commit -m "[type]: [subject]" -m "- [change 1]" -m "- [change 2]" -m "- [change 3]"
```

Or use a heredoc if preferred:
```bash
git commit -m "$(cat <<'EOF'
[type]: [subject]

- [change 1]
- [change 2]
- [change 3]
EOF
)"
```

## 7. Verify

Confirm the commit was successful by showing the commit SHA and a brief summary of what was committed.
